# 重建 Git 历史计划

## 目标

删除旧项目提交历史，只保留当前 `video-reader` 项目的文件快照作为新的首提交，避免 GitHub contributors 继续统计旧项目维护者。

## 当前判断

| 项目 | 判断 |
| --- | --- |
| contributor 来源 | GitHub 根据 Git 提交历史统计 |
| 旧 contributor 出现原因 | 仓库保留了旧项目的提交历史 |
| 处理方式 | 使用 orphan 分支重建历史 |
| 推送方式 | 使用 `--force-with-lease` 推送 `main` |

## 执行步骤

| 顺序 | 操作 | 验证 |
| --- | --- | --- |
| 1 | 检查当前工作区状态 | 确保没有未提交源码改动 |
| 2 | 创建 orphan 分支 | 新分支没有父提交 |
| 3 | 暂存当前非忽略文件快照 | `data/`、日志、缓存不进入提交 |
| 4 | 创建新的首提交 | 提交作者为当前 Git 用户 |
| 5 | 将分支命名为 `main` | 本地 `main` 指向新历史 |
| 6 | `--force-with-lease` 推送远程 | 远程 `main` 替换为新历史 |
| 7 | 验证远程提交历史 | `git shortlog -sne --all` 不再显示旧作者 |

## 风险边界

| 风险 | 控制 |
| --- | --- |
| 远程历史被替换 | 用户已明确要求删除旧 contributor 历史 |
| 运行数据误提交 | 依赖 `.gitignore`，推送前检查暂存区 |
| 远程并发更新 | 使用 `--force-with-lease` |
