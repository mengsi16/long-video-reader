# video-reader 项目整理计划

## 目标

把当前项目从旧子目录形态整理成 `video-reader` 根目录项目，更新 README，写好 `.gitignore`，迁移 Git 仓库位置，并完成一次提交。

## 当前判断

| 项目 | 判断 |
| --- | --- |
| Git 位置 | 当前 `.git` 在旧子目录内，根目录 `video-reader` 不是 Git 仓库 |
| README 状态 | 仍包含旧项目名称、克隆说明、推广内容、联系信息，不符合当前项目定位 |
| 应提交内容 | 源码、静态资源、配置模板、脚本、项目文档、必要图片、计划文档 |
| 不应提交内容 | 本地数据库、转录内容、视频帧、运行日志、临时样式文件、截图、Windows `NUL` 文件、虚拟环境、缓存、密钥文件 |

## 执行步骤

| 顺序 | 操作 | 验证方式 |
| --- | --- | --- |
| 1 | 迁移 `.git` 到 `video-reader/.git` | `git rev-parse --show-toplevel` 输出项目根目录 |
| 2 | 把旧子目录内的项目文件移动到根目录 | 根目录可见 `backend`、`static`、`start.py`、`requirements.txt` |
| 3 | 写根目录 `.gitignore` | `git status --short` 中不出现运行数据、日志、截图、临时文件 |
| 4 | 重写 `README.md`，保留当前功能和启动方式，移除旧项目推广与克隆叙述 | README 标题和结构指向 `video-reader` |
| 5 | 删除旧包装目录和不该提交的根目录临时文件 | 旧子目录不再作为项目外壳存在 |
| 6 | 运行轻量冒烟检查 | Python 编译检查通过，Git 状态只包含应提交内容 |
| 7 | 执行一次 Git 提交 | `git log -1 --oneline` 显示本次整理提交 |

## 提交边界

| 类别 | 处理 |
| --- | --- |
| 项目源码 | 提交 |
| README 与计划文档 | 提交 |
| `.gitignore`、`.dockerignore`、`.env.example`、Docker 文件 | 提交 |
| `data/`、`server.*.log`、截图和临时文件 | 忽略或删除工作区散落副本 |
| `.env`、虚拟环境、缓存目录 | 忽略 |

## 备注

本次不改业务接口和核心逻辑，只做项目根目录、文档、忽略规则和版本控制边界整理。
