# GitHub 远程上传计划

## 目标

检查当前项目能否上传到 `git@github.com:mengsi16/long-video-reader.git`，在安全条件满足时把本地提交推送到远程仓库。

## 判断顺序

| 顺序 | 检查 | 处理 |
| --- | --- | --- |
| 1 | 本地 Git 状态 | 确认是否存在未提交修改 |
| 2 | 当前分支与最近提交 | 确认要推送的分支和提交 |
| 3 | 远程仓库可访问性 | 使用 `git ls-remote` 检查 SSH 权限和远程是否为空 |
| 4 | 远程配置 | 设置或更新 `origin` 到目标仓库 |
| 5 | 提交本轮 README/API 文档调整 | 若工作区有应提交内容，先提交 |
| 6 | 推送 | 只做普通 push，不做 force push |

## 安全边界

| 情况 | 处理 |
| --- | --- |
| 远程为空 | 推送本地 `main` |
| 远程已有不同历史 | 不强推，先报告 |
| SSH 无权限 | 不改认证配置，报告错误 |
| 工作区只有运行产物 | 不提交运行产物 |
| 工作区有 README/API 文档修改 | 提交后再推送 |

## 验证

| 检查 | 命令 |
| --- | --- |
| 本地状态 | `git status --short` |
| 远程提交 | `git ls-remote origin` |
| 最近提交 | `git log -2 --oneline` |
