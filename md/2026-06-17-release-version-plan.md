# 版本发布计划

## 目标

为当前 `long-video-reader` 发布第一个版本 tag，并推送到 GitHub 远程仓库。

## 版本策略

| 方案 | 判断 |
| --- | --- |
| 日期版本，如 `2026.06.17` | 适合每日构建、数据集、文档快照 |
| 语义化版本，如 `v0.1.0` | 适合应用、库、工具项目，便于表达兼容性和功能阶段 |
| 本项目建议 | 使用语义化版本，首个可用版本为 `v0.1.0` |

## 执行步骤

| 顺序 | 操作 | 验证 |
| --- | --- | --- |
| 1 | 确认工作区干净 | `git status --short --branch` |
| 2 | 确认无已有 tag | `git tag --list` |
| 3 | 创建 annotated tag `v0.1.0` | `git tag -a v0.1.0 -m "v0.1.0"` |
| 4 | 推送 tag | `git push origin v0.1.0` |
| 5 | 验证远程 tag | `git ls-remote --tags origin v0.1.0` |

## 发布说明

| 版本 | 内容 |
| --- | --- |
| `v0.1.0` | 重构后的 long-video-reader 首个可用版本，包含本地视频阅读、转写、关键帧、索引、对话和 FFmpeg 配置说明 |
